# DV3 — Voice Companion with Emotional Visualizer

A voice-activated home companion that listens for a wake word, conducts natural conversation via the Gemini Live API, and drives a fullscreen ambient visualizer that responds to emotional context extracted from the conversation.

**Platform**: Ubuntu/WSL2 on Windows NUC
**Status**: Active Development — voice pipeline operational, web editor shipped, animation library migration in progress

---

## How It Works

```
[Wake Word] → [Gemini Live API] → [Emotion Parser] → [Visualizer]
                    |
                [Tools: Spotify, Timers, System]
```

1. Wake word detection (OpenWakeWord, local CPU) gates the Gemini Live session
2. Gemini Live handles VAD, STT, LLM reasoning, and TTS in a single WebSocket session
3. Every Gemini text response begins with a bracketed emotion tag: `[curious] That's interesting...`
4. The emotion parser extracts the tag, triggering a crossfade to the matching animation
5. Contextual triggers (music genre, weather terms) override emotion animations
6. A Pygame visualizer plays animated WebP files fullscreen with a radial black gradient mask

## Setup

- Python 3.12+
- Node.js 18+ (for web editor)
- PulseAudio or PipeWire bridge (WSL2 audio)
- Wake word model: `*.onnx` file configured in `settings.yaml`

```bash
cd ~/projects/dv3
```

## Web Editor (DV3 Editor)

The browser-based animation editor is the primary tool for building the DV3 animation library. Located at `editor-web/`, runs at `http://localhost:5173`.

**Workflow:**

1. **Import** — Use the Import button to load GIF, WebP, PNG, or JPG files into the Inbox
2. **Edit** — Apply non-destructive adjustments (brightness, contrast, hue, flip, crop, speed, reverse, etc.)
3. **Tag** — In the Tag Panel, select emotions, states, custom tags, theme, title, and notes
4. **Save** — Bakes edits via FFmpeg WASM and writes the final WebP + manifest entry to `data/animations/library/`
5. **Library** — Switch to the Library tab to re-edit, duplicate, or remove saved assets
6. **Export** — Download a ZIP of all library assets + `manifest.json` for backup or transfer

See [editor-web/README.md](editor-web/README.md) for full documentation.

## Web Visualizer

The web visualizer (`visualizer-web/`) is the default display mode — a fullscreen browser-based animation player that connects to the Python backend via WebSocket.

```bash
cd visualizer-web
npm install
npm run dev     # → http://localhost:5174
```

**Features:**
- Fullscreen black canvas with centered animation and radial gradient overlay
- Crossfade transitions between animations (300ms default)
- Idle rotation — cycles through random emotions when no backend is connected
- Emotion/state/contextual animation switching via WebSocket events from the companion
- Browser mic capture (16 kHz PCM) with echo cancellation, streamed to backend
- TTS audio playback from Gemini via WebSocket

**Settings panel** — press **S** or **Ctrl+Shift+V** to toggle:
- **Gradient Opacity** (0–100%) — controls the radial vignette darkness
- **Gradient Size** (0–100%) — how far inward the gradient extends
- **Animation Size** (20–100%) — scale of the animation relative to viewport

**Status indicators** (top-right corner):
- **wake** (amber pulse) — flashes when wake word detected
- **mic** (red pulse) — browser microphone active
- **live** (green dot) — WebSocket connected to backend

The visualizer works standalone (shows animations with idle rotation) or connected to the Python companion for live voice interaction.

## Configuration

All runtime settings are in `config/settings.yaml`. No sensitive values belong here — those go in `.env`.

Key settings:
- `wake_word.model_path` — path to the `.onnx` wake word model
- `voice.backend` — `gemini_live` (only active backend)
- `voice.model` — Gemini model ID
- `voice.conversation_idle_timeout` — seconds of silence after turn complete before returning to IDLE (default 8)
- `visualizer.fullscreen` / `target_fps` / `animation_height_pct`
- `visualizer.gradient.opacity` / `size`
- `animations.base_dir` — root of the animation library (`data/animations`)
- `visualizer_ws_port` — WebSocket server port (default 8765)

Secrets in `.env`:
- `GEMINI_API_KEY` / `GEMINI_API_KEY_FALLBACK`
- `SPOTIPY_CLIENT_ID` / `SPOTIPY_CLIENT_SECRET` / `SPOTIPY_REDIRECT_URI`

## Emotion System

Emotion detection runs a cascade:

1. **Contextual triggers** — match user speech against patterns in `emotion_map.yaml` (music genres, weather, etc.)
2. **Regex tag** — extract `[emotion]` from first 30 tokens of Gemini's text response
3. **Keyword fallback** — scan full response for sentiment keywords
4. **Default** — `neutral`

Contextual triggers override emotion triggers. Priority: contextual > emotion > keyword > neutral.

All emotion/state/contextual definitions live in `config/emotion_map.yaml`.

---

## Animation Library

Animations are managed via the web editor and stored as:

- `data/animations/library/` — flat directory of baked WebP files
- `data/animations/manifest.json` — index with full metadata (emotions, states, tags, theme)

The `EmotionMapper` class reads `manifest.json` at startup and resolves emotion queries against it, with fallback chains defined in `emotion_map.yaml`.
